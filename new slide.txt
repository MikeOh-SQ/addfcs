SLIDE 1. 표지

화면 제목
ADDFCS.COM
주의·집중·실행 기능 패턴을 정리하는 자기점검 도구

화면 보조 문구
자가보고, 수행 과제, 실행 계획, 반복 활동을 하나의 흐름으로 연결한 모바일 웹 서비스

발표 설명
이 서비스는 사용자가 최근 일상에서 느끼는 주의·집중·실행 기능 관련 어려움을 정리하고,
간단한 수행 과제와 실행 계획을 통해 자신의 패턴을 이해하도록 돕는 자기점검 도구입니다.
의학적 진단을 대신하는 서비스가 아니라, 현재 상태를 돌아보고 다음 행동을 정하는 참고 도구로 설계되었습니다.

SLIDE 2. 프로젝트 배경
==================================================

화면 제목
왜 이 서비스를 만들었는가?

화면 핵심 문구
“집중이 안 된다”는 체감만으로는
어떤 상황에서, 어떤 방식으로 어려움이 나타나는지 설명하기 어렵습니다.

화면 포인트
- 스스로 느끼는 어려움은 중요하지만 개인의 해석에 영향을 받음
- 짧은 수행 상황에서 실제 반응 패턴이 다르게 나타날 수 있음
- 결과를 알아도 오늘 무엇을 바꿀지 연결하기 어려움

발표 설명
주의·집중 또는 실행 기능의 어려움은 한 가지 질문이나 점수만으로 충분히 설명되지 않습니다.
사용자가 느끼는 부담, 과제에서 나타나는 반응, 일상에서 적용할 행동 전략을 함께 연결해야
결과가 단순한 판정이 아니라 자기이해와 실천으로 이어질 수 있습니다.

SLIDE 3. 서비스의 핵심 원칙
==================================================

화면 제목
진단이 아니라, 패턴을 이해하기 위한 자기점검

화면 포인트
- 사용자가 직접 느끼는 일상 어려움을 수집한다.
- 짧은 수행 과제로 반응 패턴을 보조적으로 확인한다.
- 두 결과를 함께 해석한다.
- 사용자가 실천할 수 있는 작은 계획으로 연결한다.
- 연구 분석을 위해 식별성이 낮은 사용 기록과 수행 흐름을 축적한다.

발표 설명
서비스 전반에서 가장 중요한 원칙은 진단을 단정하지 않는 것입니다.
결과 문구는 “확정”이 아니라 “경향”, “부담”, “보조 참고”라는 표현을 중심으로 구성됩니다.
수면, 스트레스, 기기 환경, 당시 컨디션 등 다른 요인이 영향을 줄 수 있기 때문입니다.

- “연구 논문 확인하기” 링크

기록 원리
- 신규 사용자 기록은 database/<사용자ID>-<생성시각>.json 으로 저장
- 기록에는 생성 시각, 최근 수정 시각, 현재 진행 단계가 포함
- 과거 운영 기록은 database/old/ 로 분리되어 신규 이어하기 대상에서 제외

발표 설명
처음부터 연구 목적과 데이터 활용 범위를 짧게 고지하고, 논문 링크를 별도 창에서 확인할 수 있게 했습니다.
이후 생성되는 기록은 새 데이터군으로 관리되며,
이전 기록은 별도 보관 폴더에 이동해 현재 서비스의 이어하기 목록과 분리했습니다.

SLIDE 7. 간단 자기점검
==================================================

화면 제목
1차 입력: 최근 6개월의 체감 패턴을 빠르게 확인

화면 포인트
- 총 6문항
- 0점 “전혀 그렇지 않다”부터 4점 “매우 자주 그렇다”까지 선택
- 집중 유지·일 시작 관련 4개 영역
- 반응 조절 관련 2개 영역

계산 원리
- 1~3번 문항: 2점 이상이면 부담 응답으로 계산
- 4~6번 문항: 3점 이상이면 부담 응답으로 계산
- 부담 응답 수, 집중 유지 점수, 반응 조절 점수를 저장
- 부담 응답이 4개 이상이면 부담이 비교적 크게 나타난 것으로 설명

발표 설명
간단 자기점검은 사용자가 느끼는 일상 어려움을 짧게 포착하는 시작 단계입니다.
이 결과만으로 진단하지 않고, 다음 수행 과제 및 세부 점검과 함께 해석하기 위한 주관적 입력으로 사용합니다.

SLIDE 8. 세부 자기점검
==================================================

화면 제목
2차 입력: 어느 영역의 부담이 더 두드러지는지 세분화

화면 포인트
- 총 23문항, 그렇다/아니다 응답
- 집중 유지 영역 9문항
- 반응 조절 영역 9문항
- 지속성, 여러 환경에서의 반복, 일상 영향 등을 보는 추가 확인 5문항

계산 원리
- 집중 유지 9문항 중 6개 이상 체크: 집중 유지 부담 패턴
- 반응 조절 9문항 중 6개 이상 체크: 반응 조절 부담 패턴
- 두 영역 모두 6개 이상: 복합 실행 기능 패턴
- 어느 영역도 기준에 모이지 않으면 큰 부담이 낮은 흐름으로 설명
- 추가 확인 응답은 일상 부담 수준 해석을 보강

발표 설명
세부 자기점검은 단순한 전체 점수가 아니라, 어떤 유형의 부담이 더 크게 느껴지는지 구조화합니다.
추가 확인 문항은 해당 패턴이 오래 지속되었는지, 여러 환경에서 나타나는지,
실제 생활 기능에 영향을 주는지 생각해보도록 돕습니다.

SLIDE 9. 반응성 과제를 함께 넣은 이유
==================================================

화면 제목
체감과 수행은 같을 수도, 다를 수도 있다

화면 비교
주관적 보고
- 내가 일상에서 느끼는 집중과 조절의 어려움

수행 기반 관찰
- 제한된 시간 안에서 목표를 찾는 방식
- 눌러야 할 때 누르고 멈춰야 할 때 멈추는 방식
- 움직임 또는 안정 상태를 유지하는 방식

발표 설명
사용자는 평소 어렵다고 느껴도 짧은 과제에서는 안정적으로 반응할 수 있고,
반대로 큰 어려움을 인식하지 못해도 수행 중 목표 놓침이나 억제 실패가 나타날 수 있습니다.
따라서 이 서비스는 설문과 수행을 경쟁시키지 않고, 서로 다른 관점의 자료로 교차 확인합니다.

SLIDE 10. 반응성 과제 1: 신호 찾기
==================================================

화면 제목
지속주의: 목표를 놓치지 않고 찾아내는가?

과제 방식
- 목표 자극에만 반응하고 비목표 자극에는 반응하지 않음
- 연습 후 본 시행 60회 진행
- 자극 사이 간격을 섞어 예측 반응을 줄임

핵심 저장 지표
- omission_rate: 목표를 놓친 비율
- reaction_time_variability: 반응시간의 흔들림
- tau: 유난히 늦은 반응의 꼬리
- late_phase_drop: 후반부 정확도 저하

발표 설명
신호 찾기는 정답 개수만 보는 것이 아니라, 지속적으로 주의를 유지할 수 있었는지 확인합니다.
목표를 놓친 비율과 반응시간의 변동이 커질수록
집중 유지가 순간적으로 끊기거나 흔들리는 가능성을 보조적으로 해석합니다.

==================================================
SLIDE 11. 반응성 과제 2: 멈춤 버튼
==================================================

화면 제목
반응 억제: 행동해야 할 때와 멈춰야 할 때를 구분하는가?

과제 방식
- Go 자극은 눌러야 함
- No-Go 자극은 누르지 않아야 함
- 연습 후 본 시행 50회 진행
- 너무 이른 반응은 별도로 구분

핵심 저장 지표
- commission_rate: 누르면 안 되는 자극을 누른 비율
- fast_error_rate: 성급한 반응 비율
- mean_go_reaction_time: Go 자극 평균 반응시간
- premature_response_count: 성공 기준보다 빠른 반응 수

발표 설명
멈춤 버튼은 빠른 것이 항상 좋은 결과가 아님을 보여주는 과제입니다.
빠르게 반응하는 동시에 멈춰야 할 상황에서 오류가 반복되면,
반응 억제와 충동 조절의 부담을 참고 지표로 볼 수 있습니다.

SLIDE 12. 반응성 과제 3: 균형 유지
==================================================

화면 제목
활동성 보조 지표: 안정 상태를 얼마나 유지하는가?

과제 방식
- 30초 동안 안정 상태를 유지
- 모바일 센서가 가능하면 기기 움직임 활용
- 센서 사용이 어렵다면 롱터치 또는 포인터 방식으로 대체

핵심 저장 지표
- stable_duration_pct: 안정 유지 시간 비율
- spike_count: 큰 흔들림 횟수
- total_movement: 전체 움직임 크기
- input_source: 센서, 터치, 포인터 중 실제 입력 방식

발표 설명
활동성 지표는 기기와 입력 환경의 영향을 크게 받을 수 있습니다.
따라서 이 과제는 과잉행동을 확정하는 도구가 아니라,
안정 유지와 자기조절을 바라보는 보조 정보로만 사용합니다.

SLIDE 13. 통합 리포트의 작동 원리
==================================================

화면 제목
결과는 한 점수가 아니라 두 관점의 비교로 제시

리포트 구성
- 내가 느끼는 일상: 간단·세부 자기점검 기반
- 과제에서 보인 수행: 반응성 과제 지표 기반
- 두 신호의 관계: 일치 / 불일치 / 혼합
- 집중 유지 탭: 목표 놓침, 반응시간 변동성, Tau, 후반 저하
- 반응 조절 탭: 잘못된 반응, 성급 반응, Go 반응시간
- 일상 피로도: 추가 맥락을 포함한 1~5 수준 표현

발표 설명
최종 리포트는 “당신은 어떤 유형이다”라고 단정하기보다,
자가보고에서 나타난 부담과 과제에서 보인 반응이 비슷한지 또는 다른지 설명합니다.
두 결과가 다르다면 오류로 취급하지 않고,
피로, 긴장, 환경 차이 등에 따라 나타날 수 있는 의미 있는 차이로 안내합니다.

SLIDE 14. 계산과 AI의 역할 분담
==================================================

화면 제목
숫자는 규칙으로, 설명은 AI로 보조

구조
사용자 입력
  ↓
브라우저에서 응답·수행 데이터 수집
  ↓
서버가 저장 JSON으로 핵심 지표 계산
  ↓
규칙 기반 리포트 및 계획 기본안 생성
  ↓
AI가 이해하기 쉬운 표현과 계획 조정을 보조

화면 핵심 문구
AI가 없는 값을 추정해 판정하는 구조가 아닙니다.

발표 설명
핵심 지표와 리포트 골격은 저장된 숫자에 기반한 결정론적 계산으로 생성됩니다.
AI는 간단·세부 결과를 자연스럽게 설명하거나,
실행 계획을 사용자의 생활 상황에 맞게 조정하는 역할을 담당합니다.
따라서 생성 문장과 측정값의 역할을 분리해 해석의 투명성을 확보하려고 했습니다.

SLIDE 15. 실행 계획
==================================================

화면 제목
결과를 “이해”에서 “행동”으로 연결

화면 포인트
- 리포트에 맞춰 3가지 작은 행동 계획 제시
- 사용자가 생활 맥락에 맞게 계획을 대화로 조정 가능

계획 분기
- 집중 유지 부담: 과제 쪼개기, 시각적 단서, 방해 요소 줄이기
- 반응 조절 부담: 즉각 반응 전에 멈춤, 심호흡, 짧은 메모
- 복합 흐름: 시작 장벽 낮추기와 충동 조절을 함께 적용
- 안정 흐름: 잘 되는 루틴을 지속하고 변화 조건을 기록

발표 설명
사용자가 자기 패턴을 알아도 구체적인 행동으로 연결되지 않으면 서비스 경험은 결과 화면에서 끝납니다.
이 서비스는 부담 유형별로 일상에서 실행 가능한 크기의 계획을 제안하고,
채팅을 통해 너무 어렵거나 맞지 않는 계획을 현실적인 형태로 조정하도록 구성했습니다.

==================================================
SLIDE 16. DTx 숲: 반복 참여 경험
==================================================

화면 제목
일회성 검사 이후, 실행을 이어가는 숲 경험

화면 구조
DTx 숲 허브
  ├─ PLAN 활동: 제안된 목표를 실천하고 완료 기록
  ├─ BUILD 활동: 선택 반응 미니 과제
  └─ CHOP 활동: 반응 억제 미니 과제

발표 설명
메인 점검이 사용자의 현재 패턴을 이해하는 과정이라면,
DTx는 결과 이후에도 사용자가 계획과 짧은 활동에 다시 참여하도록 만드는 반복 경험입니다.
애드와 루멘 캐릭터, 숲 배경 변화, 포인트 누적을 통해
행동 계획을 단순 텍스트가 아니라 지속 가능한 상호작용으로 확장합니다.

SLIDE 17. DTx 점수와 성장 원리
==================================================

화면 제목
숲은 수행 기록에 따라 성장한다

점수 원리
- PLAN: 목표 카드 완료 1회당 +10점
- 같은 목표는 완료 후 60분이 지나야 다시 완료 가능
- BUILD: 총 20회 수행에서 정답 수만큼 포인트 추가
- CHOP: 총 20회 수행에서 정답 수만큼 포인트 추가
- 모든 점수는 dtx.totalScore에 합산

숲 단계
- 0점 이상: stage1
- 200점 이상: stage2
- 400점 이상: stage3
- 600점 이상: stage4
- 800점 이상: stage5

스토리 원리
- 최초 진입 및 100점 단위 도달 시 애드와 루멘의 숲 이벤트가 한 번씩 표시

발표 설명
DTx의 점수는 자기점검 결과 점수와 혼동되지 않습니다.
이는 사용자의 반복 참여와 활동 완료를 보여주는 경험 포인트입니다.
숲의 배경 변화와 단계별 스토리는 수행을 시각적 성장 경험으로 전환합니다.

SLIDE 18. 기록 데이터의 구조
==================================================

화면 제목
하나의 사용자 기록에 무엇이 저장되는가?

저장 묶음
- 기본 정보: id, createdAt, updatedAt, currentStep
- 자기점검: 간단 자기점검 응답 요약, 세부 자기점검 응답 요약

- DTx: 숲 점수, 단계, 활동별 점수, 튜토리얼 완료 상태, 목표 완료 상태
- 연구 로그: 접속 세션, 체류 시간, 주요 행동 이벤트

화면 핵심 문구
한 번의 기록이 “응답 → 해석 → 계획 → 반복 참여”의 흐름을 연결합니다.

발표 설명
사용자별 JSON 기록은 단순 설문 답안지가 아니라,
서비스에서 발생한 주요 상태를 이어주는 중심 데이터입니다.
이어하기 기능은 이 기록을 다시 불러와 현재 단계와 활동 상태를 복원합니다.

SLIDE 19. 연구용 사용 로그

화면 제목

화면에는 개인을 직접 식별할 수 없는 데이터가 서비스 개선 및 연구 목적으로 활용될 수 있음을 고지합니다.
향후 분석에서는 접근 통제, 최소 수집, 비식별화, 연구 윤리 검토가 함께 필요합니다.

SLIDE 20. 기록의 안정성과 이어하기 원리
==================================================

화면 제목
메인 점검과 DTx가 같은 기록을 안전하게 공유

문제 상황

- 메인 화면이 오래된 상태를 저장하면 최신 DTx 점수가 사라질 수 있음

현재 처리 원리
- DTx 저장 요청은 DTx가 작성한 데이터임을 구분
- 메인 화면 저장은 기존 dtx, planGame, tutorials 값을 덮어쓰지 않도록 보호
- DTx에서 허브로 돌아올 때 최신 기록을 다시 조회하여 점수 표시
- 연구 로그는 여러 저장 요청 사이에서도 세션·이벤트를 병합하여 유지

발표 설명
한 사용자 기록을 여러 화면이 동시에 업데이트할 수 있기 때문에,
어느 영역이 어떤 데이터를 책임지는지 분리하는 것이 중요합니다.
현재는 메인 자기점검과 DTx 진행 상태의 저장 충돌을 막고,
화면 전환 후에도 사용자가 실제 저장된 최신 점수를 확인하도록 보완했습니다.

화면 제목

화면 제목

화면 포인트

- 연구 활용 시 개인정보 보호와 윤리 검토가 필요

발표 설명
서비스의 모든 점수는 맥락과 함께 해석되어야 합니다.
특히 DTx의 포인트는 숲 성장과 참여 유도를 위한 값이지,
주의·집중 문제의 심각성을 뜻하는 지표가 아닙니다.
어려움이 지속되거나 생활 기능 저하가 크다면 전문적인 평가를 별도로 고려해야 합니다.
